Durable Memory

Apache Ignite memory-centric platform is based on the Durable Memory architecture that allows storing and processing data and indexes both in memory and on disk when the Ignite Persistent Store is enabled.

The memory architecture helps achieve in-memory performance with durability of disk using all the available resources of the cluster.

The durable memory is built and operates in a way similar to the Virtual Memory of operating systems such as Linux. However, one significant difference between these two types of architectures is that Durable Memory always keeps the whole data set and indexes on disk, if the Ignite Persistent Store is used, while Virtual Memory uses the disk for swapping purposes only.

These are the advantages and characteristics Apache Ignite as a platform gains relying on the Durable Memory and Ignite Persistent Store:

In-Memory
  • Off-Heap memory
  • Removes noticeable GC pauses
  • Automatic Defragmentation
  • Predictable memory consumption
  • Boosts SQL performance
On Disk
  • Optional Persistence
  • Support of flash, SSD, Intel 3D Xpoint
  • Stores superset of data
  • Fully Transactional
    • Write-Ahead-Log (WAL)
  • Instantaneous Cluster Restarts
