Start Contributing

Before you start working on Apache Ignite, you should sign up to the dev mailing list and say "Hello" to everyone in the community. We have a very friendly group and intend to make joining the project as easy as possible.

Note, you need to subscribe to the dev list, before sending emails to it. To subscribe, send an empty email to dev-subscribe@ignite.apache.org and follow simple instructions in the reply.

Ignite Jira:

You should also sign up to Ignite Jira and browse through open tickets.

Development Process:

Before you start contributing, it helps to familiarize yourself with Ignite Development Process.

To Get Started:
  • Choose any of the tickets below.
  • Send an email to the dev list letting us know that you want to work on it (make sure to subscribe first).
  • One of the committers will add you to the contributor list and assign the ticket to you.

Start Developing

Apache Ignite uses GIT for its VCS system. The "master" branch contains the last released version. The "sprint-N" branch contains the latest code for the Nth sprint.

GIT checkout:
                # Master branch.
                $ git clone https://git-wip-us.apache.org/repos/asf/ignite

                # Release branch.
                $ git clone https://git-wip-us.apache.org/repos/asf/ignite -b ignite-1.3.0
Build Project:
                # Build project.
                $ mvn clean package -DskipTests

                # Run Apache RAT
                $ mvn clean validate -Pcheck-licenses

See more detailed build instructions in the DEVNOTES.txt file.

To Start Contributing:
  • Pick a ticket from Ignite Jira
  • Create a branch in GIT (same name as the ticket, all lowercase)
  • When ready, submit a patch in Jira and assign to a committer
  • Send email to the dev list stating that patch is ready
  • The committer will provide comments after review.
  • If the patch is rejected, address comments and repeat the process.

Become a Committer

The most important point of becoming a committer for Apache projects is active participation within the community. Being an active community member means participating on mailing list discussions, helping to answer questions, being respectful towards others, and following the meritocratic principles of community management.

Of course, having a few code contributions is also important, but it is not as important as active community participation. Hence, the minimal requirement to become a committer is to have at least one code contribution that passed the committer review process and was merged to the Ignite code base.

Candidates for new committers or PMC members are suggested by the current committers, mentors, or PMC members, and then must be approved by the PMC voting process.
