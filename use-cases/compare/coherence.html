Ignite vs Coherence

Both, Apache Ignite and Oracle Coherence provide a feature rich data grid functionality responsible for partitioning and caching data in memory with the capability to scale out across distributed clusters.

Ignite and Coherence have many differences in the way caching, transactions, and data querying are supported. Below we outline only major differences, which in our opinion are most important whenever choosing an in-memory data grid product.

Apache Ignite Coherence
Open Source vs. Closed Source

Ignite is an Apache project and has been growing and enhancing its open source feature set. The latest major additions include Ignite Fabric for C++ and .NET/C#, with Node.JS integration coming soon as well.

Coherence is a proprietary Oracle software and does not have a free open source offering.

JCache (JSR 107)

Ignite is fully compliant with JCache (JSR 107) specification for caching.

Coherence is fully compliant with JCache (JSR 107) specification for caching.

Off-Heap Memory

Ignite supports storing data in on-heap or off-heap memory, depending on user configuration.

Oracle Coherence does provide developers with a limited option to store the data in off-heap memory.

Off-Heap Indexes

Whenever configured with off-heap memory, Ignite will store query indexes off-heap as well (in order not to affect on-heap memory used by user applications).

Coherence does not have support for off-heap indexes.

SQL Queries

Ignite supports complete SQL (ANSI-99) syntax for querying in-memory data.

Coherence does not support SQL.

Queries with JOINs

Ignite SQL includes full support for SQL JOINs, including JOINs across multiple caches, for example:

   select * from A a, B b where a.b_id = b.id

Coherence does not have any support for JOIN queries (with or without SQL). Whenever needed, users perform JOINs by manually combining multiple query results.

ACID Transactions

Ignite transactions provide excellent performance with 100s of thousands transactions/sec per server.

Coherence does not recommend using transactions due to its poor performance.

Tiered Storage

Ignite provides tiered storage model where data can be stored and moved between on-heap, off-heap, and swap space. Going up the tier provides more data storage capacity, with gradual increase in latency.

Coherence does not have any support for tiered storage.

Data Streaming

Ignite provides support for in-memory streaming, including support for maintaining and querying sliding windows of streaming data.

Coherence does not offer any support for streaming.

Configuration

Ignite supports standard Java beans for configuration and natively integrates with Spring XML, as well as provides ability to easily configure the system directly from code.

Coherence uses a proprietary XML format for configuration and does not have any support for configuring a system directly from code.
