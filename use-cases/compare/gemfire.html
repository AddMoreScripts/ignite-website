Ignite vs Gemfire

Both, Apache Ignite and Pivotal Gemfire provide a feature rich data grid functionality responsible for partitioning and caching data in memory with the capability to scale out across distributed clusters.

Ignite and Gemfire have many differences in the way caching, transactions, and data querying are supported. Below we outline only major differences, which in our opinion are most important whenever choosing an in-memory data grid product.

Apache Ignite Gemfire
Open Source vs. Closed Source

Ignite is an Apache project and has been growing and enhancing its open source feature set. The latest major additions include Ignite Fabric for C++ and .NET/C#, with Node.JS integration coming soon as well.

Gemfire is a proprietary Pivotal software.

JCache (JSR107)

Ignite data grid is an implementation of JCache (JSR 107) specification. JCache provides a very simple to use, but yet very powerful API for data access.

Gemfire does not implement JCache (JSR 107) and uses proprietary APIs.

On-Heap vs. Off-Heap

Ignite supports storing data in on-heap or off-heap memory, depending on user configuration.

Gemfire does not offer any support for storing data off-heap.

SQL Queries

Ignite supports complete SQL (ANSI-99) syntax for querying in-memory data.

Gemfire does not support the standard SQL syntax. Instead it comes with its own object query language, called OQL.

Queries with JOINs

Ignite SQL includes full support for SQL JOINs, including JOINs across multiple caches, for example:

   select * from A a, B b where a.b_id = b.id

Gemfire does not have any support for JOIN queries across caches or regions. Whenever needed, users perform JOINs by manually combining multiple query results.

Cross-Partition Transactions

Ignite supports cross-partition transactions where transactions can be performed on all partitions of a cache across the whole cluster.

Gemfire does not support transactions across multiple cache partitions or nodes.

Tiered Storage

Apache Ignite provides tiered storage model where data can be stored and moved between on-heap, off-heap, and swap space. Going up the tier provides more data storage capacity, with gradual increase in latency.

Gemfire does not have any support for tiered storage.

Data Streaming

Ignite provides support for in-memory streaming, including support for maintaining and querying sliding windows of streaming data.

Gemfire does not offer any support for streaming.

Configuration

Ignite supports standard Java beans for configuration and natively integrates with Spring XML, as well as provides ability to easily configure the system directly from code.

Gemfire uses a proprietary XML format for configuration and does not have any support for configuring a system directly from code.

Deployment

Ignite nodes are peers and automatically join the cluster on startup (without any locator servers).

Gemfire requires a locator service to be started and maintained in order for nodes to join or leave the cluster.
