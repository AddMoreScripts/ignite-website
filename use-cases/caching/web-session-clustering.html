Ignite for Web Session Clustering

Failing-over web session caching is problematic when you run multiple application servers. Most web applications run in a cluster to distribute the load of high volume of web requests. When one of the application servers crashes, the load balancer will just route the web request to another available application server, but all of user’s session data is lost. A feasible solution here would be to cache all your web sessions in Ignite cache.

Apache Ignite is capable of caching web sessions of all Java Servlet containers that follow Java Servlet 3.0 Specification, including Apache Tomcat, Eclipse Jetty, Oracle WebLogic, and others.

Ignite Web Sessions Cache is a distributed cache that maintains a copy of all web sessions’ data in memory. In case of an application server failure, web requests get routed to some other application server that simply fetches the web session data from Ignite cache.


Ignite In-Memory Data Fabric ensures fault tolerance by either replicating or partitioning the sessions' data across all grid nodes in the cluster. Hence, no session is ever lost, providing better user experience. Furtermore, with web session caching, sticky connections become less important as the session is available on any application server the web request may be routed to.

Apache Ignite Web Session clustering helps your web application to:

  • Scale out your application servers horizontally as the load increases.
  • Achieve fault tolerance by configuring number of backups across the data grid.
  • Easily configure all types of application servers in plug-n-play fashion.
