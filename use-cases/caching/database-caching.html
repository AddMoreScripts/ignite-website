Ignite for Database Caching

Ignite in-memory data grid is a key-value in-memory store which enables caching data in-memory within distributed clusters.

Ignite database caching enables users to keep the most frequently accessed data in memory, by either partitioning or replicating it across a cluster of computers.

In addition to being highly elastic and resilient, Ignite is lightning fast and a fully compliant implementation of JCache (JSR 107) specification. Ignite is also fully compliant with other caching technologies, including Memcached, Spring and Hibernate.

Ignite Data Grid:
Database caching is supported by the Data Grid component.

Ignite Features for Database Caching

Feature Description
Key-Value Store

Ignite data grid is an in-memory key-value store which can be viewed as a distributed partitioned hash map, with every cluster node owning a portion of the overall data. This way the more cluster nodes we add, the more data we can cache.

Write-Through & Read-Through

Whenever an update happens in the in-memory cache, Ignite can optionally propagate it to the underlying database. Also, whenever the data is requested from cache and is not there, Ignite can automatically load it from the underlying database. Such behavior is called write-through and read-through correspondingly.

Docs for this Feature
Write-Behind

In a simple write-through mode each cache put and remove operation will involve a corresponding request to the persistent storage and therefore the overall duration of the cache update might be relatively long. To mitigate such cases, Ignite offers an option to perform an asynchronous persistent store update also known as write-behind.

Automatic Persistence

Ignite ships with its own database schema mapping wizard which provides automatic support for integrating with persistence stores.

Docs for this Feature
Pluggable Persistence

Ignite supports pluggable persistence semantics specified by JCache specification.

Docs for this Feature
ACID Transactions

Ignite in-memory cache is fully consistent and fully transactional. Whenever pluggable persistence is turned on, database and in-memory cache operations are automatically merged into one overall transaction.

Queries & Distributed Joins

Ignite supports a very elegant query API with support for:

  • Scan Queries (Predicate-based)
  • SQL Queries (ANSI 99)
  • Text Queries

For SQL and TEXT queries ignites supports in-memory indexing, so all the data lookups are extremely fast. If you are caching your data in off-heap memory, then query indexes will also be cached in off-heap memory as well.

Ignite also allows users to implement their own custom indexing using pluggable IndexingSpi.
