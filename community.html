Apache Ignite Community Resources

This section contains resources to help the community get started with Apache Ignite.

Development1

Apache Ignite uses GIT for its VCS system. The "master" branch contains the last released version. The "sprint-N" branch contains the latest code for the Nth sprint. 

                # Master branch.
                git clone https://git-wip-us.apache.org/repos/asf/incubator-ignite

                # Sprint branch.
                git clone https://git-wip-us.apache.org/repos/asf/incubator-ignite -b sprint-2
To start contributing:
  1. Pick a ticket from Ignite Jira
  2. Create a branch in GIT from the latest sprint branch. If ticket is IGNITE-1234 then GIT branch name should be ignite-1234
  3. Whenever the branch is ready for review, create a patch, attach it to the Jira ticket, and assign to any of the committers (sending email to the DEV list also helps)
  4. The committer will provide comments after review.
  5. If the patch is accepted, then the committer will apply the patch and close the ticket.
  6. If the patch is rejected, the contributor should address the comments and submit a new patch (do not delete previous patches from tickets).
