Distributed Events

Developers can use distributed events to get notified about remote tasks executions or any cache data changes within the grid. With GridGain distributed events support you can

  • Subscribe local and remote listeners within grid
  • Enable and disable any event
  • Provide local and remote filters for fine-grained control over notifications

Subscribe remote and local notifications for cache data updates

                                         try (Grid grid = GridGain.start("examples/config/example-cache.xml")) {
                                             final GridCache<Integer, String> cache = g.cache(CACHE_NAME);

                                             // Sample remote filter to accept only keys that are greater than 10.
                                             GridPredicate<GridCacheEvent> rmtLsnr = new GridPredicate<GridCacheEvent>() {
                                                @Override public boolean apply(GridCacheEvent evt) {
                                                   println("Cache event: " + evt);

                                                   int key = evt.key();

                                                   // Accept only keys greater than 10.
                                                   return key >= 10;
                                                }
                                             };

                                             // Subscribe to specified cache events on all nodes that have cache running.
                                             grid.forCache(CACHE_NAME).events().remoteListen(null, rmtLsnr,
                                                 EVT_CACHE_OBJECT_PUT,
                                                 EVT_CACHE_OBJECT_READ,
                                                 EVT_CACHE_OBJECT_REMOVED).get();
                                         }
